# PNG to GLTF Converter

一个将 PNG 图片离线转换为 GLTF 3D 模型文件的工具。该工具可以将 2D 图片转换为带深度信息的 3D 模型，适合用于制作钥匙链、浮雕等 3D 打印或可视化项目。

## 功能特点

- 🖼️ **图片转 3D 模型**：将 PNG 图片转换为 GLTF 格式的 3D 模型
- ⚙️ **可配置参数**：支持自定义宽度、高度、深度、细分等参数
- 🔄 **保持宽高比**：自动保持原图宽高比，避免模型变形
- 💾 **多种输出格式**：支持 GLTF（.gltf + .bin + 纹理）和 GLB 格式
- 🌐 **离线转换**：使用浏览器技术，无需在线服务
- 🎨 **纹理支持**：保留原始图片作为模型纹理

## 技术栈

- **Node.js**：运行环境
- **Playwright**：浏览器自动化，用于执行转换逻辑
- **Three.js**：3D 图形处理和渲染
- **自定义库**：
  - `imageObject.min.js`：图片转 3D 对象
  - `exporter.min.js`：模型导出
  - `GLTFExporter.js`：GLTF 格式导出
  - `jszip.min.js`：ZIP 文件打包

## 安装

1. 确保已安装 Node.js（推荐 v14 或更高版本）

2. 安装依赖：

```bash
npm config set registry https://registry.npmmirror.com
npm install
```

3. 安装 Playwright 浏览器（如果尚未安装）：

```bash
npx playwright install chromium
```

## 使用方法

### 1. 准备输入图片

将需要转换的 PNG 图片放在项目根目录，命名为 `yiyi5.png`（或修改脚本中的图片路径）。

### 2. 配置转换参数

编辑 `transfer_params.txt` 文件，配置转换参数：

```
工具：标准(颜色)
宽度：400
宽度细分：400
高度：400
高度细分：400
边框：不勾选
深度：15
合并相似颜色：5
透明度转换：黑色
保存格式：gltf
锁定纵横比：勾选
逆变输出：不勾选
启用平滑：不勾选
Z轴反射镜：不勾选
```

**参数说明**：
- **工具**：转换工具类型（目前支持：标准(颜色)）
- **宽度/高度**：模型尺寸（像素）
- **宽度细分/高度细分**：网格细分程度，影响模型精度
- **边框**：是否添加边框
- **深度**：模型的深度（Z 轴方向）
- **合并相似颜色**：颜色合并阈值，用于优化模型
- **透明度转换**：如何处理透明区域（黑色/其他）
- **保存格式**：输出格式（gltf 或 glb）
- **锁定纵横比**：是否保持原图宽高比
- **逆变输出**：是否反转模型
- **启用平滑**：是否启用平滑处理
- **Z轴反射镜**：是否启用 Z 轴镜像

### 3. 运行转换

```bash
node test_convert_playwright.js
```

或者使用 npm 脚本：

```bash
npm test
```

### 4. 获取输出

转换完成后，输出文件将保存为 `output.gltf.zip`，包含：
- `yiyi5.gltf`：GLTF 模型文件
- `yiyi5_0.bin`：二进制数据文件
- `textures/image_0.png`：纹理图片

## 工作原理

1. **启动本地 HTTP 服务器**：在本地 8080 端口启动服务器，提供资源文件访问
2. **加载图片**：读取 PNG 图片并转换为 Base64 数据 URL
3. **创建 HTML 页面**：动态生成包含转换逻辑的 HTML 页面
4. **浏览器执行**：使用 Playwright 启动 Chromium 浏览器，加载页面
5. **3D 处理**：
   - 使用 Three.js 创建场景和渲染器
   - 使用 `ImageObject` 将图片转换为 3D 几何体
   - 根据深度参数填充几何体，创建实心 3D 模型
   - 添加底面和侧面，形成完整模型
6. **导出模型**：使用 GLTFExporter 将模型导出为 GLTF 格式
7. **打包输出**：将 GLTF、BIN 和纹理文件打包为 ZIP

const pngPath = path.join(__dirname, 'your_image.png');

在脚本中找到以下代码并修改：

```javascript
const bufferFileName = 'yiyi5_' + index + '.bin';

```

### 优化模型质量

- **提高细分**：增加 `宽度细分` 和 `高度细分` 值（注意：会显著增加文件大小）
- **调整深度**：根据图片内容调整 `深度` 值
- **颜色合并**：调整 `合并相似颜色` 来平衡模型质量和文件大小

## 注意事项

1. **文件大小**：高细分会产生较大的 GLTF 文件，脚本中已包含顶点采样优化以控制文件大小
2. **浏览器内存**：处理大图片时，可能需要调整浏览器内存限制
3. **透明度处理**：透明区域会被转换为黑色背景，可通过 `透明度转换` 参数调整
4. **模型方向**：生成的模型可能需要根据实际需求旋转或调整方向

## 常见问题

### Q: 转换失败怎么办？

A: 检查以下内容：
- PNG 文件是否存在且可读
- 端口 8080 是否被占用
- Playwright 浏览器是否已正确安装
- 查看控制台输出的错误信息

### Q: 生成的模型有黑点或空洞？

A: 可以尝试：
- 调整 `合并相似颜色` 参数
- 修改脚本中的 `alphaTest` 阈值
- 检查原图的透明度处理

### Q: 如何减小输出文件大小？

A: 可以：
- 降低宽度和高度细分值
- 调整脚本中的 `topVertexSampleRate` 采样率
- 压缩输入 PNG 图片
